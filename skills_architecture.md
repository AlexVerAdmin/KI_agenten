# Архитектура системы "Skills-First Framework" для AntigravityAgents

## 1. Концепция
Переход от монолитного управления через LLM к модульной системе **Скиллов (Skills)**. 
Цель: Повышение стабильности локальных моделей (Mistral Nemo 12B/Llama 3.1 8B), исключение галлюцинаций при вызове системных команд и оптимизация нагрузки на GPU (P4000).

---

## 2. Уровни логики (Layers)

### Уровень 1: Маршрутизация (Router/Planner)
- **Триггер**: Перехватывает ввод пользователя (Regex + классификация моделью).
- **Логика**: Если запрос соответствует известному "скиллу", управление передается напрямую в `SkillManager`, минуя генерацию JSON моделью.
- **Интеллект**: Модель используется только для понимания намерения (Intent Recognition) в коротком контексте.

### Уровень 2: Банк Скиллов (Skill Bank)
- **Структура**: Папка `core/skills/`, содержащая атомарные Python-классы.
- **Атомарность**: Каждый скилл отвечает за одну область (GPU, VDS, Docker, Obsidian).
- **Скилл "GPU_Inspector"**:
    - Инкапсулирует IP воркера (88.55).
    - Выполняет `nvidia-smi`.
    - Форматирует ответ в читаемый вид.
- **Скилл "VDS_Safety_Shell"**:
    - Реализует текстовый HITL (Human-in-the-Loop).
    - Сначала спрашивает разрешение, затем вызывает `subprocess` или SSH.

### Уровень 3: Уровень Рассуждений (Reasoning Layer)
- Задействуется только для:
    1. Интерпретации результатов скиллов (например: "Почему место на диске кончилось?").
    2. Сложных диалогов (Немецкий язык, Карьера).
    3. Создания новых скиллов (Meta-Skill).

---

## 3. Специализированные Скиллы (Мета-уровень)

- **Skill-Designer (Мета-скилл)**:
    - Позволяет системе самой писать код для новых скиллов на основе документации.
    - Требует песочницы для тестирования сгенерированного кода.
- **Architect-Skill (Планировщик)**:
    - Декомпозирует сложную цель (например: "Перенеси контейнер с VDS на локальный сервер") на цепочку атомарных скиллов.

---

## 4. Схема взаимодействия (Data Flow)

1. **User Input** -> "Как там поживает моя видеокарта?"
2. **Router** -> Распознан паттерн `[GPU_STATUS]`.
3. **SkillManager** -> Загружает `skills/gpu_skill.py`.
4. **Execution (Worker)** -> HTTP запрос на `192.168.88.55:8001/execute`.
5. **Output Processing** -> Текстовый результат из `nvidia-smi`.
6. **LLM (Reasoning)** -> (Опционально) "Видеокарта в порядке, температура 50°C".

---

## 5. Преимущества
- **Надежность**: Нулевая вероятность галлюцинации кода при выполнении системных задач.
- **Скорость**: Мгновенный ответ для стандартных запросов без ожидания генерации токенов.
- **Безопасность**: Жесткая изоляция опасных команд внутри проверенных скиллов.
