# Структура проекта Antigravity Agents

Данный проект представляет собой систему многоагентного ИИ с пользовательским интерфейсом на Streamlit и Telegram-ботом.

## Основные компоненты

### Корневая директория
- [app.py](app.py) — Основной файл веб-интерфейса на базе Streamlit. Позволяет переключаться между агентами, просматривать историю чатов и отправлять сообщения.
- [bot.py](bot.py) — Telegram-бот, реализованный с использованием библиотеки `aiogram`. Поддерживает голосовые сообщения (транскрипция через Groq Whisper).
- [config.py](config.py) — Конфигурация проекта, управление ключами API (Groq, Google Gemini и др.) и настройками.
- [requirements.txt](requirements.txt) — Список зависимостей Python.
- [run_ui.bat](run_ui.bat) / [run_bot.bat](run_bot.bat) — Скрипты для быстрого запуска веб-интерфейса и Telegram-бота соответственно.
- [memory_v2.sqlite](memory_v2.sqlite) — Основная база данных SQLite для хранения истории чатов и настроек агентов.

### Директория `core/` (Ядро системы)
- [core/orchestrator.py](core/orchestrator.py) — Основная логика управления агентами. Использует `LangGraph` для построения графов состояний агентов. Включает:
    - Регистрацию агентов (`AGENT_REGISTRY`).
    - Сохранение и получение истории чатов из SQLite.
    - Интеграцию с LLM (Google Gemini, Groq).
- [core/orchestrator_v2.py](core/orchestrator_v2.py) — Вторая версия оркестратора (вероятно, экспериментальная или расширенная).
- [core/memory.py](core/memory.py) — Модуль для управления памятью агентов.

### Директория `utils/` (Утилиты)
- [utils/audio_utils.py](utils/audio_utils.py) — Функции для работы со звуком: преобразование текста в речь (TTS) и обработка аудио для Telegram-бота.

### Директория `chroma_db/`
Хранилище векторной базы данных Chroma, используемое для RAG (Retrieval-Augmented Generation) и долгосрочной памяти агентов.
