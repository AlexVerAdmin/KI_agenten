# План реализации голосового общения в реальном времени (Realtime Voice)

Этот документ описывает переход от текущей схемы (STT -> Text -> LLM -> TTS) к потоковому голосовому общению для агента "Учитель немецкого" на базе существующего FastAPI интерфейса.

## 1. Текущая архитектура
- **Backend:** FastAPI (`Agents/src/gateway/web.py`) на порту 8080.
- **Frontend:** HTML/JS встроенный в `web.py`.
- **Процесс:** Браузерный `Web Speech API` (STT) -> WebSocket (текст) -> LLM -> TTS (`edge-tts` или Gemini TTS) -> WebSocket (путь к файлу) -> Воспроизведение.
- **Проблема:** Задержка из-за последовательных шагов и использования браузерного STT, который требует тишины для завершения фразы.

## 2. Целевая архитектура (Gemini 3.1 Flash Live)
Использование **Gemini 3.1 Flash Live** (Multimodal Live API) для прямой передачи аудио-потока.

### Технологический стек:
- **Backend:** FastAPI WebSocket эндпоинт `/ws/realtime/{agent}`.
- **Протокол:** Двусторонний WebSocket (Binary/JSON) для передачи PCM аудио (16kHz).
- **Модель:** `gemini-2.0-flash-exp` (или актуальная 3.1 Flash Live).
- **Frontend:** Обновленный JS-клиент с использованием `AudioWorklet` для захвата микрофона без задержек.

## 3. Новые функции управления (в /settings)
Для комфортного обучения немецкому языку добавляются следующие настройки:
1.  **Voice Activation Mode:**
    *   `Always On`: Микрофон постоянно слушает (режим рации).
    *   `Push-to-Talk`: Активация только при нажатии.
    *   `Auto-Pause`: Автоматическое включение микрофона после ответа учителя.
2.  **Silence Threshold (Пауза):** Настройка времени ожидания (в мс) перед тем, как модель решит, что пользователь закончил говорить.
3.  **Speech Rate (Скорость речи):** Регулировка темпа речи учителя (от 0.5x до 2.0x). Это критически важно для начинающих (A1-A2), чтобы лучше воспринимать немецкую речь на слух.
4.  **Voice Toggle:** Глобальный переключатель режима "Realtime Voice" в интерфейсе.

## 4. Этапы реализации

### Этап 1: Backend (FastAPI)
- Создание нового роутера в `src/gateway/web.py` для обработки бинарных аудио-данных.
- Интеграция с Google Generative AI SDK (Multimodal Live Session).
- Проброс системного промпта из `german_teacher.py`.

### Этап 2: Frontend (JS)
- Добавление переключателя "Realtime Mode" рядом с кнопкой микрофона.
- Реализация захвата аудио через `navigator.mediaDevices.getUserMedia`.
- Отправка аудио-фреймов через WebSocket.
- Обработка входящего аудио-потока от Gemini и его немедленное воспроизведение.

### Этап 3: Настройки (/settings)
- Добавление полей в `SETTINGS_HTML` для управления параметрами Realtime (чувствительность, задержки).
- Сохранение этих параметров в `memory_v2.sqlite` через существующий API настроек.

## 6. Преимущества для Учителя немецкого
- **Мгновенная реакция:** Ученик может перебивать учителя, как в реальной жизни.
- **Фонетика:** Модель слышит нюансы произношения напрямую, а не через текстовую транскрипцию.
- **Естественные паузы:** Настройка паузы позволяет ученику дольше подбирать слова, не боясь, что его прервут.
