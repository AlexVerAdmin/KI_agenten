# 🏗 Архитектура Скиллов (Knowledge Handshake & Infrastructure)

## 1. Концепция: Разделение Инструкций и Данных
Вместо перегрузки System Prompt сложными правилами форматирования (YAML, пути, теги), мы переходим к **Skill-Centric** архитектуре.
- **Model (Intellect):** Решает *ЧТО* сделать (сохранить слово, проверить ресурсы).
- **Skill (Python Logic):** Решает *КАК* это сделать (форматирование, пути, протоколы).

---

## 2. Управление Знаниями (VDS <-> Home)
### 2.1 Активная База Выжимок (Knowledge Sandbox)
На VDS не хранится весь Obsidian Vault. Доступны только актуальные **выжимки (Knowledge Fragments)**.
- **Local Model (Home Librarian):** Домашний сервер с мощной моделью выступает куратором. Он анализирует основной Obsidian и генерирует компактные `.md` файлы в папку `Sandbox/`.
- **Sync:** Содержимое `Sandbox/` синхронизируется на VDS по SSH.
- **Intelligent GC (Garbage Collection):** Локальная модель удаляет неактуальные выжимки из `Sandbox/`, когда тема освоена, обеспечивая всегда "свежий" и компактный контекст для агентов на VDS.

### 2.2 Механизм "Маячков" (Knowledge signals)
Если агенту на VDS не хватает данных для ответа (дефицит контекста):
1. **Signal:** Агент вызывает скилл `signal_knowledge_gap(topic, query)`.
2. **Trigger:** На VDS в папке `/inbox/` создается микро-файл (JSON-маячок) с описанием пробела в знаниях.
3. **Response:** Локальная модель дома при сканировании видит маячок, делает `semantic_search` по основному Obsidian и "пробрасывает" нужную выжимку на VDS по SSH.
4. **Data Isolation:** История диалогов хранится локально на VDS в SQLite. Домой передаются только итоговые отчеты и маячки.

## 3. Библиотека Скиллов (Skills Library)

### 3.1 Obsidian Skills (Универсальные)
- `skill_read_fragment(path)`: Чтение файлов из локальной папки знаний агента на VDS.
- `skill_save_to_inbox(content, tags)`: Запись новых данных (слова, расходы) в папку VDS `/inbox/` для последующего забора домашним сервером.
- `skill_signal_gap(topic, query)`: Создание маячка для запроса новых знаний у домашнего "Библиотекаря".

### 3.2 Infrastructure Skills (Контроль Ресурсов)
Универсальный набор инструментов для контроля серверов.
- **Targeting:** Скилл принимает параметр `target` (`local` или `vds`).
- **Functionality:**
    - `get_system_status(target)`: Мониторинг CPU, RAM, Disk.
    - `manage_docker_container(target, name, action)`: Управление контейнерами (Start/Stop/Restart).
    - `check_network_latency(target)`: Проверка связи между узлами для диагностики сети.

## 5. Безопасность и Изоляция
- Агенты на VDS имеют доступ только к своим изолированным папкам выжимок (`/german/`, `/career/`, `/finance/`).
- Прямой доступ к основному Obsidian Vault закрыт; взаимодействие идет через SSH-мостик и папку Inbox.
- Входящие документы из VDS обрабатываются домашней моделью перед интеграцией в основной Vault.
5. **Output Processing** -> Текстовый результат из `nvidia-smi`.
6. **LLM (Reasoning)** -> (Опционально) "Видеокарта в порядке, температура 50°C".

## 5. Преимущества
- **Надежность**: Нулевая вероятность галлюцинации кода при выполнении системных задач.
- **Скорость**: Мгновенный ответ для стандартных запросов без ожидания генерации токенов.
- **Безопасность**: Жесткая изоляция опасных команд внутри проверенных скиллов.
