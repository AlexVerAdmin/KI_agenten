# Правила разработки и деплоя (CI/CD)

## 1. Написание кода
- Весь новый функционал должен быть задокументирован в `Agents/plans/`.
- Код должен соответствовать существующей архитектуре (FastAPI, WebSocket, SQLite).
- После реализации обязательна проверка на синтаксические ошибки и базовое тестирование.

## 2. Процесс выгрузки (Deployment Workflow)
При завершении задачи агент должен следовать строгому алгоритму:

1. **Проверка:** Выполнить проверку реализованного функционала.
2. **Запрос разрешения:** Спросить пользователя: *"Задача реализована и проверена. Выгрузить изменения на GitHub и обновить VDS?"*
3. **GitHub Push:** После получения разрешения выполнить:
   ```bash
   git add .
   git commit -m "feat: [описание задачи]"
   git push origin main
   ```
4. **VDS Update:** Подключиться к VDS (или использовать автоматику) для загрузки кода:
   ```bash
   cd /path/to/project && git pull origin main
   ```
5. **Restart Service:** При необходимости перезагрузить сервис или пересобрать Docker-контейнеры:
   ```bash
   docker-compose up -d --build
   ```

## 3. Параметры исполнения
- **Mode:** `code` или `orchestrator`.
- **Target Environment:** VDS (Linux).
- **Port:** 8080 (Web UI).
